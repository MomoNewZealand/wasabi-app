# 🌾 わさび田管理システム - セットアップマニュアル

## 📋 概要

このシステムは、わさび栽培の「植え付け」「生育」「出荷」「加工」を一元管理するWebアプリです。

- 🌐 **スマホブラウザ** で URL を開くだけで使用可能
- 📱 **PWA化** でアプリみたいにホーム画面に追加可能
- 🔐 **Google アカウント** でセキュアにログイン
- 📊 **ダッシュボード** で出荷量や歩留まり率を可視化

---

## 🚀 セットアップ手順（5ステップ）

### **ステップ 1️⃣：Supabase でテーブルを作成**

1. **Supabase ダッシュボード** を開く
   - https://app.supabase.com

2. **「Wasabi Project」** をクリック

3. 左メニューの **「SQL Editor」** をクリック

4. **「+ New Query」** をクリック

5. 以下のSQLを貼り付ける（`01_create_tables.sql` の内容）
   ```sql
   -- テーブル作成スクリプルをここに貼り付け
   CREATE TABLE IF NOT EXISTS users (
     ...
   )
   ```

6. **「Run」** をクリックして実行

   ✅ 成功したら、「Tables」に以下が表示されます：
   - `users`
   - `destinations`
   - `plantings`
   - `shipments`
   - `processing`

---

### **ステップ 2️⃣：初期データを投入**

1. もう一度 **「+ New Query」** をクリック

2. 以下のSQLを貼り付ける（`02_insert_data.sql` の内容）
   ```sql
   -- データ投入スクリプトをここに貼り付け
   INSERT INTO plantings (location, planted_date, ...) VALUES
     ...
   ```

3. **「Run」** をクリック

   ✅ 成功したら、データが以下に投入されます：
   - `plantings`: 21行
   - `processing`: 8行
   - `destinations`: 5個（六雁、佐ノ家、Satologue、The Flats、わさび食堂）

---

### **ステップ 3️⃣：API キーを取得**

1. **Settings → API** をクリック

2. **「Project URL」** をコピー
   ```
   例: https://xxxxxxxxxxxx.supabase.co
   ```

3. **「anon key」** をコピー
   ```
   例: eyJhbGc...（長い文字列）
   ```

4. どこかにメモしておく（ステップ 4 で必要）

---

### **ステップ 4️⃣：環境変数を設定**

> 📌 このステップは「開発者向け」です。最終的にはホストされたアプリで提供します。

ローカルでテストしたい場合：

1. `.env.local` ファイルを作成
   ```
   REACT_APP_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
   REACT_APP_SUPABASE_ANON_KEY=eyJhbGc...
   ```

2. 保存

---

### **ステップ 5️⃣：Google OAuth を設定（オプション）

> 簡単なので、できればやっておくと楽です。

1. **Supabase → Settings → Authentication → Providers**

2. **「Google」** を見つける → **「Enable」** をクリック

3. Google Cloud Console で設定が必要
   - （複雑なので、後で説明します）

   省略の場合：**メールアドレス/パスワード認証** でも使用可能

## 💻 アプリをデプロイ（Vercel）

### **方法A：私が代わりにデプロイ**（推奨）

```
1. ファイルをGitHubにアップロード
2. Vercelと連携
3. URLをあなたに配布
→ その日のうちに完成
```

### **方法B：あなたでデプロイ**（高度）

1. **GitHub** にコードをプッシュ

2. **Vercel** にサインアップ
   - https://vercel.com

3. **「New Project」** → GitHub リポジトリを選択

4. 環境変数を設定
   - `REACT_APP_SUPABASE_URL`
   - `REACT_APP_SUPABASE_ANON_KEY`

5. **「Deploy」** をクリック

   ✅ 数分で URL が生成されます！

## 📱 スマホでの使い方

### **初回：ブラウザで開く**

1. 配布された URL をスマホのブラウザで開く
   ```
   例: https://wasabi-app.vercel.app
   ```

2. **Google でログイン**

### **2回目以降：ホーム画面に追加**

```
iOS（iPhone）:
  1. ブラウザの「共有」をタップ
  2. 「ホーム画面に追加」
  3. 完了

Android:
  1. メニュー（3点）→ 「ホーム画面に追加」
  2. または ブラウザ右上「...」→ 「アプリをインストール」
```

以後、ホーム画面からアプリのようにタップして開けます。

## 🔧 トラブルシューティング

### **Q: ログインできない**
```
A: Google OAuth を有効にしてください（ステップ 5）
   それでも駄目なら、メール/パスワード認証に切り替え可能
```

### **Q: データが見えない**
```
A: Supabase の Row Level Security（RLS）が有効になっているか確認
   Settings → Authentication → Policies で確認
```

### **Q: スマホから見づらい**
```
A: ブラウザの「拡大を縮小」するか、
   PWA化（ホーム画面追加）で改善します
```

### **Q: 出荷記録で植え付けロットが選べない**
```
A: 先に「植え付け記録」タブでロットを登録してください
```

## 📊 ダッシュボード機能詳細

### **月別出荷量**
- 各月の総出荷量（g）を折れ線グラフで表示
- トレンド把握に最適

### **出荷先別出荷量**
- 出荷先ごとの合計出荷量を棒グラフで表示
- 常連先 vs 新規客の把握

### **部位別加工歩留まり率**
- 花、茎・葉ごとの平均歩留まり率
- どの部位が効率的か把握可能

### **植え付けから出荷までの日数**
- 植え付けから実際の出荷まで何日かかったか
- 品種別・エリア別の生育速度比較が可能

### **出荷先ごとの頻度**
- 各出荷先との取引頻度
- ビジネス分析に活用

### **部位別歩留まり比較**
- 部位ごとの歩留まり率を詳細に表示
- 加工効率改善のヒント
